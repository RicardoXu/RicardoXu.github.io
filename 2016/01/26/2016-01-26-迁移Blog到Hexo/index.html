迁移Blog到Hexo.md

这学期开始有个想法是在 GitHub 上搭一个自己的博客，一开始的想法是使用 Jekyll ，后来由于要做其他项目于是只做了简单的文章发布功能，同时界面实在谈不上好看。临近过年，终于稍微清闲一些，偶然发觉 Hexo 更容易上手并且效果不错，于是就把博客的内容迁移过来了。

这里只简单记录一些 Hexo 写文章和发布的命令，配置以及其他教程可以 Google 或者访问 Hexo 的文档

前置工作

关联 Hexo 和 GitHub 之前在 Github 添加ssh

在 Git Bash 执行命令：

ssh-keygen

生成文件 id_rsa.pub ，将文件内容添加到 GitHub 帐号，执行命令：

ssh -T git@github.com

检测是否联通。

写文章

新建文章命令：

$ hexo new [layout] <title>

例如：

$ hexo new "Hello World.md"

发布

Hexo 提供了快速方便的一键发布功能，只需一条命令就能将网站部署到服务器上。

$ hexo generate --deploy

不过在开始之前，需要先安装hexo-deployer-git：

$ npm install hexo-deployer-git --save

同时在 _config.yml 中修改配置：

deploy:
    type: git
    repo: <repository url> #库（Repository）地址
    branch: [branch] #可空缺，GitHub 可自动检测
    message: [message] #可空缺，自定义提交信息
