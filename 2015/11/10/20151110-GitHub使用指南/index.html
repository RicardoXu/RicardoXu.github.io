GitHub使用指南

发表于 2015-11-10 |

这是一份不完备的Ubuntu下使用Github的指南。

Ubuntu平台

Ubuntu 15.10

安装

sudo apt-get install git

sudo apt-get install git-core

配置

生成ssh

ssh-keygen -C "email" -t rsa

登陆GitHub，记录ssh，检查结果。

ssh -v git@github.com

这里还需要绑定一下自己的账号。

Windows平台

在Git Bash执行命令，生成文件id_rsa.pub：

ssh-keygen

将文件内容添加到GitHub帐号之后，执行命令，检测是否联通：

ssh -T git@github.com

并且执行命令，关联账号：

$ git config --global user.name "yourname"
$ git config --global user.email  "youremail@email.com"

Git操作

新建本地仓库

切换到新建好的仓库目录下，执行命令

git init

上传github

往暂存区域添加已添加和修改的文件，不处理删除的文件，.代表全部添加，针对特定文件可直接输入文件名：

git add .

比较本地数据目录与暂存区域的变化，在实际操作中可以省略：

git status

提到代码到本地git，并添加提交说明：

git commit -m "commit directions"

关联到github：

git remote add origin git@github.com:UserName/Respository

作为master分支上传：

git push -u origin master

从github下拉仓库

git clone https://……

分支管理

本地分支

创建分支并转到：

git checkout -b branchname

该命令等价于：

git branch branchname
git checkout branchname

查看当前分支：

git branch

之后就可以在当前分支上正常add、commit。

如果需要切换分支，使用命令：

git checkout branchname

需要把其他分支的成果合并到当前分支时，使用命令：

git merge otherbranchname

合并完成后可以删除其他分支，使用命令：

git branch -d otherbranchname

远程分支

将一个本地分支推送到远程作为某一分支：

git push origin localbranch：remotebranch

删除某个远程分支：

git push origin :newbranch

注意在:的左边是没有字符的，这代表了删除。

标签

打标签只需要切换到目标分支上，使用命令：

git tag tagname

也可以给之前的版本打标签，使用命令：

git tag -a tagname 7位校验和

查看所有标签需要使用命令：

git tag

查看标签信息：

git show tagname

删除标签：

git tag -d tagname

通常push不会将标签提交到git服务器，需要显式的操作提交单个或者全部的标签：

git push origin tagname
git push origin --tags

查看历史记录

显示前3条log信息，简单的哈希值，作者，提交时间，提交说明：

git log --pretty=format:"%h %an %ar %s" -3

以上。
