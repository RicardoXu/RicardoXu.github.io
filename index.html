好了现在让我们回归题目吧。

初试

完美户型

第一题是要求在n*m的01矩阵里寻找（至少2个）且（十字相邻）的权值是1的块，并且块的周围（米字相邻）只能是0，输出满足这些条件的块的数目。

我的思路是这样的：遍历全图为1的结点，做一遍染色（dfs），找到2个以上十字相邻的块，从2开始依次将他们的权值重置，记录块数num；第二次遍历，对于编号>1的块，考察他们周围结点的权值，一旦出现不为0或本身值得结点，则中断对该点的考察，num-1。最后输出num即可。

查找二叉树

第二题是对于一个二叉树，计算它的root到每一个leaf的权值之和中最小的值。

很典型的dfs遍历。需要注意的是，对于NULL结点，可以在进入之前就进行一次if判断，存在就继续遍历，不存在就返回0，不需要自己定义奇怪的返回值（QAQ我就NC的写了个返回-1什么的，结果还要在上一层判断写的好麻烦）。

简单总结基础算法的思想

最近报了个信息学比赛，想把脑子里那些只剩下名字的算法捡一下，就写了这么个稿子，只有算法思想，不包含具体实现。

最小生成树

Prim

在图中任选一点作为初始生成树集合，找到从集合出发的权值最小边加入生成树，最小边的点加入集合，更新集合与剩余点的权值，再次寻找新的最小边，重复该过程直至所有点都进入生成树集合。

最短路径

Spfa

以源点S为初始队列，一维数组dist【v】是源点S到其余点的距离。从队列中取出首个结点，经过该点松弛dist【v】，如果松弛成功，则将成功的点加入队列，重复该过程直至队列为空。

排序

QuickSort

选择位于中间数值作为判断标准，从两端开始将大于/小于该值的数值进行交换，直到不能进行为止，这保证了排序后的结果是以中间数值为标准左小右大的。之后从最左端到当前位置，从当前位置到最右端递归该过程。

迁移Blog到Hexo.md

这学期开始有个想法是在GitHub上搭一个自己的博客，一开始的想法是使用Jekyll，后来由于要做其他项目于是只做了简单的文章发布功能，同时界面实在谈不上好看。临近过年，终于稍微清闲一些，偶然发觉Hexo更容易上手并且效果不错，于是就把博客的内容迁移过来了。

这里只简单记录一些Hexo写文章和发布的命令，配置以及其他教程可以Google或者访问Hexo的文档

前置工作

关联Hexo和GitHub之前需要在Github添加ssh

在Git Bash执行命令：

ssh-keygen

生成文件id_rsa.pub，将文件内容添加到GitHub帐号，执行命令：

ssh -T git@github.com

检测是否联通。

并且执行命令：

$ git config --global user.name "yourname"
$ git config --global user.email "youremail@email.com"

写明自己身份。

写文章

新建文章命令：

$ hexo new [layout] <title>

例如：

$ hexo new "Hello World.md"

发布

Hexo提供了快速方便的一键发布功能，只需一条命令就能将网站部署到服务器上。

$ hexo generate --deploy

不过在开始之前，需要先安装hexo-deployer-git：

$ npm install hexo-deployer-git --save

同时在 _config.yml 中修改配置：

deploy:
    type: git
    repo: <repository url> #库（Repository）地址
    branch: [branch] #可空缺，GitHub 可自动检测
    message: [message] #可空缺，自定义提交信息

Shadowsocks使用指南

Shadowsocks是我比较喜欢的一套win下科学上网方式，客户端很简洁，节点也比较稳定，这里记录了在win下和Ubuntu下的Shadowsocks使用方式。

安装

Windows7：可以直接在购买Shadowsocks的网站下载Win7对应版本的客户端，该客户端提供了全局模式和PAC模式，不需要额外的浏览器插件设置代理。

Ubuntu 15.10：运行代码：

sudo apt-get install shadowsocks

运行

Windows7：Shadowsocks的Win客户端很简明易懂，最后选择PAC模式，即自动切换代理模式即可，在这里不多介绍。

Ubuntu 15.10：配置文件，切换到/etc/shadowsocks/目录下：

sudo gedit /etc/shadowsocks/config.json

根据购买的账号修改各项参数：

  • server服务器IP(IPv4/IPv6)，注意这也将是服务端监听的IP地址

  • server_port服务器端口

  • local_port本地端端口

  • password用来加密的密码

  • timeout超时时间（秒）

  • method加密方法，可选择“bf-cfb”,“aes-256-cfb”,“des-cfb”,“rc4”,等等。默认是一种不安全的加密，推荐用“aes-256-cfb”。

保存退出，运行：

sudo sslocal –c /etc/shadowsocks/config.json

至此，命令行配置完成，在浏览器设置好代理配置即可，FireFox 代理插件推荐Pan，还可以提供广告的过滤功能。

后台运行

切换到/etc/shadowsocks/目录下：

nohup sslocal –c /etc/shadowsocks/coig.json &

Markdown使用指南

这几天了解到一门很漂亮的文字写作格式，叫做Markdown。不过写文字和代码是方便了，但是对数学公式的支持好像并不好。这里简单记录一下Markdown的使用方法。

标题

在Markdown中，如果一段文字被定义为标题，只要在这段文字前加 # 号和空格即可。

一级标题

二级标题

三级标题

以此类推，总共六级标题。

另外，如果要在第一个字符打出 # 本身这个符号，在其前面加一个空格即可。

列表

加 * 即可变为无序列表，加 1. 2. 3. 可变为有序列表。

符号要和文字之间加上一个字符的空格。

  • a
  • b
  • c

两个列表不能相邻。

  1. a
  2. b
  3. c

在需要被嵌套的列表第一个符号前加空格可以实现嵌套。

引用

引用格式。

Example

在文本前加入 > 这种尖括号即可。

图片与链接

插入链接与插入图片的语法类似，区别是图片的话需要在最前面加入一个 ! 号。

[name](url)

粗体与斜体

用一对 * 号括起来的部分是可以转换成加斜/加粗的。

一对表示加斜，两对表示加粗，三对表示加斜加粗。

代码框

一对 ` 可以实现代码框的功能。

分割线

三个星号 * 可以添加一道分割线。

创建链接

<>

为输入的URL或邮箱自动创建链接，如test@email.com。

转义字符

转义字符 \ 在Markdown里是生效的。

使用平台

Windows: MarkdownPad2

Linux： Ubuntu

结束语

基本功能就是这些了，其他功能以后会视需要进行补充。

Vim使用指南

为了向高端Emacs党和Vim党看齐，了解了一下Vim，作为初学者记录了一些简单的用法。

OS: Ubuntu

安装： sudo apt-get install vim-gtk

配置： sudo vim /etc/vim/vimrc

在最后一行开始输入：

set nu // 在左侧行号

set tabstop=4 //tab 长度设置为 4

set nobackup //覆盖文件时不备份

set cursorline //突出显示当前行

set ruler //在右下角显示光标位置的状态行

set autoindent //自动缩进

Insert模式

按下 i 键进入Insert模式，在Insert模式中，可以进行文字的编辑。

按下Esc键返回Normal模式。

Normal模式

基本操作

  • x 删当前光标所在的个字符。
  • :wq 存盘 + 退出(:w 存盘, :q 退出)。
  • dd 删除当前行，并把删除的行存到剪贴板里。
  • p 粘贴剪贴板到当前行的下一行

修改Ubuntu开机引导项

在安装好Win和Ubuntu双系统后，有时我们会发现，开一次机需要经历Win和Ubuntu的两次开机引导，浪费了许多开机时间。

本文的目的在于消灭Ubuntu的开机引导，只保留Win的开机引导。理由是多年间习惯了Win的开机引导样式。就是这么任性。:-D

平台版本

Windows7 x64 ultimate

Ubuntu 15.10

具体操作

在Ubuntu终端执行命令：

sudo gedit /etc/default/grub

kylin：

将找到的三个GRUB_TIMEOUT=10中的else上下的两个改为GRUB_TIMEOUT=0.1，注意不可改为0。

原版：

仅有一个GRUB_TIMEOUT=10可供修改。

保存退出，执行命令：

sudo update-grub

这样就可以略过 Ubuntu 的开机引导了。
